
# Purpose
Redevelop the legacy Scale Labeling Control (SLC) application using a modern development 

# Objectives
- One-to-One feature parity with the legacy application
- A modularized application which is easily extensible.
- Support current scales and printers.
- Full localization support for English, Spanish and French
- Fully telemetry support for report remote support.  
	- Design first logging and diagnostic.
- Stand along configuration and execution without CFS sister systems.
	- [ ] What does it mean when those systems are in place?
	- [ ] What variations of disconnected operations should we support

# Architecture Statement
- Electron based web application running against
	- Svelte-Electron
	- Microsoft SQL Server

# Executing Hardware
- Existing SLC equipment
	- Windows 11 PC
	- 16GB RAM, 512GB storage drive
	- Display 1024x768
	- Single USB-A port
	- Elo touch screen display
	- Compatible scale
	- Compatible printer

# Deployment
- Docker
	- Container for Electron elements
	- Dedicated container for mssql
	- Dedicated container for Grafana and Prometheus telemetry
- Alternatively, on windows we may have to pass on the containers and do a bare metal install.	

# Unit Testing
- Implement unit testing with the Arrange, Assess, Assert pattern
- Objective: 80% code coverage
- Use Vitest
- Fully documented business rules to assist with test generation

# End to End Testing
- Playwright
- Fully documented business rules for assisted test generation.

# Existing Customer Data
- Migration utility for user level permissions to role based security in new system.
